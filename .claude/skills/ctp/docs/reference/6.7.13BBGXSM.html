◇ 1.3.订阅私有流接口调整

新增 THOST_TERT_RESUME_FROM_SEQ_NO 私有流订阅模式，支持从指定序号开始重传，若未填写序号则默认从序号1开始重传：

（1）SubscribePrivateTopic 接口，入参增加 int nSeqNo，不填默认为1；

（2）THOST_TE_RESUME_TYPE 新增枚举值：THOST_TERT_RESUME_FROM_SEQ_NO

///@param nSeqNo 即将被处理的私有流的序号

◇ 1.5.优化中继上报用户信息接口

优化中继上报用户信息接口 RegisterUserSystemInfo，新增 MAC 字段。

  • 特别说明：应期货公司按监管要求清洗数据要求，监管字段在此接口均已独立出来。请中继终端规范填写此接口的 ClientPublicIP、ClientAppID、MAC字段，另外ClientLoginRemark 字段建议填写格式为"UUID|手机号.."，注意均填写终端投资者的信息。

  • 另外，考虑到监管趋势日趋严厉，CTP后续可能会考虑将这三个字段由非必填改为必填，请提前规范填写。

///客户终端的MAC等标识
TThostFtdcDeviceTagType MAC;

◇ 1.6.新增支持郑商所套利确认和套保确认指令的接口

注意，交易所仍在盘后结算才能确认。

说明：

（1）套利确认请求、套保确认请求，若CTP校验通过后给 Rtn通知 / CTP校验未通过 则给 Rsp响应和ErrRtn —> 从交易所回来收到回报后给Rtn通知（若交易所校验未通过是没有ErrRtn的）。

（2）套利确认撤销请求、套保确认撤销请求，若CTP校验通过后给 Rtn通知 / CTP校验未通过 则给 Rsp响应和ErrRtn —> 从交易所回来收到回报后给Rtn通知（若交易所校验未通过会有ErrRtn的）。

◇ 1.10.后台V6.7.13版本优化，不涉及API修改，但对API有影响的变更：

◇ 1.10.1.支持上期能源套利和市价指令业务，API使用方式有变化，具体如下：

(1)上期能源市价指令：

价格TThostFtdcPriceType LimitPrice字段填“保护价”；

报单价格条件 TThostFtdcOrderPriceTypeType OrderPriceType字段填 “任意价”

(2)上期能源套利指令：

组合开平标志CombOffsetFlag字段 和 组合投机套保标志CombHedgeflag字段需要输入两位。

示例：CombOffsetFlag=‘04’(表示左腿开，右腿平昨仓)，CombHedgeFlag=‘13’(表示左腿投机，右腿套保)；

其他说明：CombOffsetFlag=‘01’的1平仓也会自动转成4平昨；互换单标志 IsSwapOrder字段对上期所无效，也就是说 即便两腿开平标志不同，lsSwapOrder也填false，无需填true

(3)可开可平手数计算：

市价指令，采用保护价计算保证金；

套利指令，traderapi最大可下单手数中开平标志暂不支持输入两位，请用户分别查单腿的再取最小值作为套利合约的可开可平手数。

(4)其他说明-交易所规则：

上期能源的市价、套利指令报单，交易所仅允许在合约处于“连续交易”状态时报入；

集合竞价报单阶段仅允许撤销转限价后的市价报单或套利合约报单，不允许报入市价、套利指令报单。

◇ 1.10.2.支持不指定合约申报广期所的履约对冲业务（即合约代码可不填）。
